Terminology Query Module

Current Status

Third development sprint on the Terminology Query Module in progress

The aim has been to get a hosted solution with a first draft of a REST based query interface deployed and accessible.

Following the principles, only small steps in each sprint, aiming for simplicity and clarity

Current Successes

Significant percentage of 'back-end' WB code is now hosted
based on the work done by the VA

First pass of a REST base API has been created

Both of these are now deployed to an Amazon EC2 instance

These are early days...

mistakes will be made
but lessons will be learnt,
and implementations will continue to improve.

REST URL

At its simplest, the URL looks like

http://api.snomedtools.com/query-service/query?for=x,let=x,where=x,return=x,viewpoint=x

FOR parameter




	CONCEPT

Defines the set of components to iterate over

LET parameter




  
	
	  allergic-asthma
	  
		Allergic asthma
		531abe20-8324-3db9-9104-8bcdbf251ac7

Defines references to concept specifications or other view coordinates used by where clauses

WHERE parameter




  
	allergic-asthma
	CONCEPT_IS_KIND_OF

Defines the where clauses for the query

RETURN parameter




    CONCEPT_VERSION

Defines component types to be return (Concepts, descriptions, etc)
