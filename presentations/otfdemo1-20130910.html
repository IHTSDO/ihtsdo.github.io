Terminology Query Module

Current Status

Two (and a bit) development sprints on the Terminology Query Module have been completed

The aim has been to get a hosted solution with a first draft of a REST based query interface deployed and accessible.

Following the principles, only small steps in each sprint, aiming for simplicity and clarity

Current Successes

Significant percentage of 'back-end' WB code is now hosted

First pass of a REST base API has been created

Both of these are now deployed to an Amazon EC2 instance

Details on the whole project build can be found here
http://ihtsdo.github.io/OTF-Query-Services/index.html

These are early days...
expect changes in both content and direction as more development comes online.

Java Code

The code can be cloned from GitHub with this command 


								$ git clone https://github.com/IHTSDO/OTF-Query-Services.git
then built using maven in the root directory of the repo 

								$ mvn clean install

A jar file is created which can then be executed by typing 


								java -cp query-client/target/query-client-1.1-SNAPSHOT-jar-with-dependencies.jar org.ihtsdo.otf.query.rest.client.examples.KindOfQueryExample http://api.snomedtools.com

and the result looks like this

the example java code, KindOfQueryExample, can be seen in the org.ihtsdo.otf.query.rest.client.examples
folder in the repository

REST URL

At its simplest, the URL looks like

http://api.snomedtools.com/query-service/query?for=x,let=x,where=x,return=x,viewpoint=x

FOR parameter




	CONCEPT

Defines the set of components to iterate over

LET parameter




  
	
	  allergic-asthma
	  
		Allergic asthma
		531abe20-8324-3db9-9104-8bcdbf251ac7

Defines references to concept specifications or other view coordinates used by where clauses

WHERE parameter




  
	allergic-asthma
	CONCEPT_IS_KIND_OF

Defines the where clauses for the query

RETURN parameter




    CONCEPT_VERSION

Defines component types to be return (Concepts, descriptions, etc)
