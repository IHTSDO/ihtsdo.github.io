SNOMED INTERNATIONAL: GUI GUIDELINES

This is a living document of the Snomed International GUI guidelines, with examples for how certain web elements should look or operate when producing webapps or similar programs.

All newly written applications for Snomed International should follow the same design theme provided here, most importantly with identical sized headers and footers, to allow seamless transitions between different applications.

Did you know, you can save yourself time by forking or copying this repository, and you will have the headers, footers, modals, and much more already waiting for you to use!

COLOURS

These are the majority of colours used in the Snomed International applications. Each of these colours acts as a base which can be darkened or lightened if necessary using darken() or lighten() sass functions.

For example, a delete button should have a background in MANDY, which can be darkened by 10% when hovering over it.

TYPEFACE

All typography in snomed applications is measured in the unit of rem. This stands for root em, and is way of providing uniform font sizing in web programming. The measurement of rem requires a base font size, of which ours is 12px.

This means a font of 1rem is equal to 12px, or a font of 1.5rem is equal to 18px

FORM ELEMENTS

Webapps often have a lot of form inputs, either as part of a singular form, or sometimes aggregated into a much larger application. This is considered guidance on how these buttons should typically look. These are heavily based on bootstrap.

MODALS

Our own modal service exists, in which modals can be opened and closed via uniquely named ids. In this example I have made one called example-modal.

Extra attention should be paid to preserving the look of the header and the footer in the modal, in order to maintain consistency across our applications.

TOASTR

Toastr is an included library that can be used to provide feedback to the users for a variety of reasons, a good example of which being a successful (or unsuccessful) http response.

Much like buttons, these too come with some associated types, green for success, red for failure/error.

LIBRARIES

This example, along with all the most recent webapps developed for Snomed International have been developed in Angular v12 via TypeScript, and are regularly upgraded to the latest versions.

There are a common set of libaries included within all our webapps that include, but is not limited to:

  • Font Awesome
  • Bootstrap
  • Toastr
  • RxJs
